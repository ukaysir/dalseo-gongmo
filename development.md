# 달서 AI 내 주소 영향 요약 서비스 개발계획

## 1. 서비스 개요

### 서비스명

달서 AI 내 주소 영향 요약 서비스

### 한 줄 설명

달서구의 고시공고, 도시계획, 교통, 주차, 안전, 복지, 환경 정보를 주소 기준으로 묶어 주민이 자기 생활권에 영향을 주는 변화를 쉬운 말로 확인하게 돕는 공공서비스이다.

### 현재 구현 상태

- Next.js App Router 기반 대시보드와 랜딩/서비스 설명 페이지 구현
- Supabase `impact_items`, `sources` 우선 조회 및 로컬 JSON fallback 구현
- 대구광역시 돌발 교통정보 API와 구간소통정보 API를 `/api/impacts`에 병합
- 지도 선택 위치, 현재 위치, 반경 선택, 생활 유형별 코치, 목록/지도/상세 패널 구현
- OpenAI API 기반 채팅과 리포트는 후속 단계이며, 현재 코치는 규칙 기반 생활영향 분석이다.

### 해결하려는 문제

달서구청과 달서구의회에는 고시공고, 도시관리계획, 주민의견청취, 교통통제, 공사 안내, 회의록 등 많은 행정정보가 공개되어 있다. 그러나 일반 주민 입장에서는 다음과 같은 어려움이 있다.

- 공고 제목만 보고 내 생활과 관련 있는지 판단하기 어렵다.
- 도시계획, 도로점용, 교통통제, 주민의견청취 같은 행정 용어가 어렵다.
- 내 집, 학교, 직장, 가게 주변에 어떤 영향이 있는지 알기 어렵다.
- 의견 제출 기한이나 주민설명회 일정을 놓치기 쉽다.
- 회의록과 공고문은 길고 복잡해 핵심 쟁점을 빠르게 파악하기 어렵다.

본 서비스는 공개 행정자료를 주민 주소 기준의 생활영향 정보로 바꾸어, 행정정보 접근성과 주민참여 기회를 높이는 것을 목표로 한다.

## 2. 주요 사용자

### 일반 주민

- 우리 집 주변 공사, 교통통제, 도시계획 변경, 공공사업 영향을 알고 싶은 주민
- 고시공고를 읽기 어렵거나 놓치기 쉬운 주민
- 의견 제출 기한과 방법을 쉽게 확인하고 싶은 주민

### 소상공인

- 가게 주변 도로공사, 주차 제한, 행사 통제, 상권 영향 정보를 알고 싶은 상인
- 영업시간, 배달 동선, 고객 접근성에 영향을 주는 행정정보를 미리 알고 싶은 사용자

### 학생·학부모

- 학교 주변 공사, 통학로 변경, 교통통제, 안전 관련 공고를 확인하고 싶은 사용자

### 달서구청 담당자

- 주민에게 공개된 행정정보를 더 쉽게 전달하고 싶은 담당자
- 반복 문의와 민원 발생을 줄이고 싶은 부서
- 주민의견 제출과 정책 참여율을 높이고 싶은 부서

## 3. 핵심 기능

### 3.1 내 주소 기반 영향 요약

주민이 주소를 입력하거나 지도에서 위치를 선택하면 해당 위치 주변의 행정정보와 공공 API 항목을 찾아 영향 가능성을 요약한다.

- 반경 500m, 1km, 2km 단위 관련 항목 검색
- 공사, 교통통제, 도시계획, 행사, 환경, 안전 관련 항목 분류
- 내 주소와의 거리 표시
- 예상 영향 항목 표시: 소음, 주차, 통행, 보행, 상권, 통학, 재산권, 생활편의
- 영향 기간과 의견 제출 기한 안내

### 3.2 어려운 행정문서 쉬운말 요약

고시공고와 회의록을 일반 주민이 이해할 수 있도록 바꾼다.

- 3줄 요약
- 핵심 쟁점
- 주민에게 중요한 날짜
- 의견 제출 방법
- 담당 부서와 연락처
- 어려운 행정 용어 해설

### 3.3 주민설명회·회의록 쟁점 요약

달서구의회 영상회의록, 회의록, 주민설명회 자료를 분석한다.

- 사업 목적 요약
- 찬성·우려 의견 분리
- 주민 생활 영향 항목 추출
- 앞으로 남은 절차 정리
- 내 주소와 관련된 내용만 필터링

### 3.4 의견서 초안 생성

주민이 선택한 이슈에 대해 의견 제출 문안을 작성할 수 있도록 돕는다.

- 찬성 의견서 초안
- 우려 의견서 초안
- 개선 요청 의견서 초안
- 공손하고 행정문서에 맞는 표현으로 변환
- 개인정보와 과도한 표현 점검

### 3.5 맞춤 알림

주민이 관심 주소를 등록하면 관련 공고가 새로 올라왔을 때 알림을 제공한다.

- 내 주소 주변 신규 공고 알림
- 의견 제출 마감 임박 알림
- 공사 시작·종료 예정 알림
- 교통통제 시작 전 알림
- 주민설명회 일정 알림

## 4. 활용 데이터

### 4.1 달서구청 고시공고

활용 내용:

- 도시관리계획 열람공고
- 주민의견청취 공고
- 행정예고
- 공사·용역·사업 관련 공고
- 도로점용, 교통, 환경, 안전 관련 공고

수집 방법:

- 달서구청 홈페이지 고시공고 게시판 정기 크롤링
- 게시글 제목, 작성일, 담당부서, 본문, 첨부파일 URL 수집
- HWP, HWPX, PDF, DOCX 첨부파일 텍스트 추출
- 문서 내 주소, 지번, 도로명, 기간, 담당부서, 연락처 추출

업데이트 방식:

- 1시간 또는 3시간 단위 자동 수집
- 신규 게시글 발견 시 문서 파싱 작업 실행
- 수정 게시글은 게시일·수정일·첨부파일 해시값 비교로 재처리

### 4.2 달서구 공보·입법예고·보도자료

활용 내용:

- 조례 개정
- 행정 제도 변경
- 생활 관련 정책 발표
- 주민 의견수렴 일정

수집 방법:

- 달서구청 공보, 입법예고, 보도자료 게시판 크롤링
- 제목과 본문에서 정책 분야 자동 분류
- 주민 생활 영향 가능성이 높은 문서 우선 색인

업데이트 방식:

- 매일 1회 정기 수집
- 마감일이 있는 문서는 알림 대상 데이터로 등록

### 4.3 달서구의회 회의록·영상회의록

활용 내용:

- 본회의, 상임위원회, 5분 자유발언, 구정질문
- 지역 현안, 공사, 도시계획, 복지, 교통, 안전 관련 발언
- 정책 추진 배경과 쟁점 분석

수집 방법:

- 달서구의회 회의록 검색 페이지에서 회의명, 회차, 발언자, 본문 수집
- 영상회의록은 제목, 회의일, 안건명, 자막 또는 회의록 텍스트를 우선 활용
- 자막이 없는 영상은 장기적으로 음성인식 기반 전사 기능 도입

업데이트 방식:

- 매일 1회 신규 회의록 확인
- 회의록 확정 전에는 임시 데이터로 표시
- 확정 회의록 공개 시 기존 요약 재생성

### 4.4 대구광역시 교통통제 정보

활용 내용:

- 도로공사
- 행사
- 집회
- 사고
- 교통통제 시작·종료 시간
- 통제 위치와 위경도

수집 방법:

- 공공데이터포털 또는 대구 교통정보 제공 API 활용
- 통제 유형, 주소, 좌표, 시작 시간, 종료 시간 수집
- 달서구 행정구역에 해당하는 데이터만 필터링

업데이트 방식:

- 5분에서 10분 단위 API 갱신
- 실시간성이 필요한 교통통제는 캐시 만료 시간을 짧게 설정
- 종료 시간이 지난 데이터는 자동 비활성화

### 4.5 대구교통종합정보

활용 내용:

- 돌발상황
- CCTV 위치
- 도로 소통 상태
- 교통 통제 정보 보완

수집 방법:

- 공개 API 또는 제공 가능한 데이터 연계
- 실시간 교통정보는 원문 링크와 함께 표시
- 불확실한 예측 정보는 "가능성"으로 표시하고 원본 출처 제공

업데이트 방식:

- 5분 단위 갱신
- 장애 발생 시 마지막 정상 수집 시각 표시

### 4.6 주소·공간정보

활용 내용:

- 주민 입력 주소를 좌표로 변환
- 공고문에 포함된 장소와 주민 주소 간 거리 계산
- 반경 기반 영향권 분석
- 행정동, 학교, 공원, 도로, 공공시설 주변 영향 분석

수집 방법:

- 도로명주소 API를 통한 주소 정제 및 좌표 변환
- VWorld, 행정안전부 도로명주소 API, 공공데이터포털 공간정보 활용
- 학교, 공공시설, 공원, 주차장 등은 공공데이터포털 데이터 활용

업데이트 방식:

- 주소 API는 요청 시 실시간 호출
- 행정구역·공공시설 기준 데이터는 월 1회 갱신
- 좌표 변환 결과는 캐시에 저장하여 반복 비용 절감

### 4.7 주민 제보 데이터

활용 내용:

- 공고에는 없지만 주민이 체감하는 불편 정보 보완
- 소음, 주차, 통행, 안전, 보행 불편 등 실제 영향 검증

수집 방법:

- 서비스 내 간단 제보 폼 제공
- 사진, 위치, 불편 유형, 발생 시간 수집
- 개인정보 수집은 최소화하고 익명 제보 가능

업데이트 방식:

- 실시간 접수
- AI가 유형과 위치를 자동 분류
- 허위·중복 제보 필터링 후 통계에 반영

## 5. 데이터 처리 구조

### 5.1 전체 흐름

1. 데이터 수집기가 달서구청, 달서구의회, 공공데이터 API에서 원문 데이터를 수집한다.
2. 문서 파서가 PDF, HWP, HWPX, DOCX, HTML 본문에서 텍스트를 추출한다.
3. OpenAI 연동 단계의 분석기가 장소, 기간, 담당부서, 사업유형, 주민 영향 항목을 추출한다.
4. 주소·좌표 변환기가 문서 속 장소를 지도 좌표로 변환한다.
5. Supabase와 로컬 JSON에 정규화 항목을 저장한다.
6. 사용자가 주소를 입력하면 주변 관련 문서와 공공 API 항목을 검색한다.
7. 현재 MVP는 규칙 기반으로 쉬운말 요약과 생활영향 코치를 생성한다.
8. OpenAI 연동 단계에서는 검색된 근거 문서만 바탕으로 채팅 답변과 리포트를 생성한다.
9. 결과 화면에는 요약, 원문 출처, 위치, 기간, 의견 제출 방법을 함께 표시한다.

### 5.2 AI 분석 항목

- 문서 유형: 공고, 고시, 회의록, 보도자료, 교통통제, 행사
- 정책 분야: 교통, 도시계획, 안전, 환경, 복지, 문화, 교육, 상권
- 위치 정보: 주소, 지번, 도로명, 시설명, 행정동, 좌표
- 시간 정보: 시작일, 종료일, 의견 제출 마감일, 회의일
- 영향 유형: 소음, 통행, 주차, 보행, 통학, 상권, 경관, 재산권, 안전
- 주민 행동: 열람, 의견 제출, 설명회 참석, 우회, 문의, 신고
- 담당 정보: 부서명, 전화번호, 이메일, 접수처

### 5.3 신뢰성 확보 방식

- AI 답변에는 반드시 원문 출처 링크를 표시한다.
- 원문에서 확인되지 않은 내용은 단정하지 않고 "가능성"으로 표현한다.
- 주소 기반 영향은 거리, 기간, 문서 유형을 기준으로 점수화한다.
- 법적 판단이나 최종 행정 결정은 담당 부서 확인 필요 문구를 표시한다.
- 주민 의견서 초안은 제출 전 사용자가 직접 확인하도록 안내한다.

## 6. 기술 스택

### 6.1 프론트엔드

- React 또는 Next.js
- TypeScript
- 지도: Kakao Maps API 또는 Naver Maps API
- UI: Tailwind CSS 또는 공공서비스 디자인 가이드 기반 CSS
- 주요 화면:
  - 주소 검색 화면
  - 내 주변 영향 요약 화면
  - 지도 기반 영향권 표시
  - 공고 상세 요약 화면
  - 의견서 초안 작성 화면
  - 관심 주소 알림 설정 화면

### 6.2 백엔드

- Python FastAPI 또는 Node.js NestJS
- REST API 또는 GraphQL API
- 인증: 휴대폰 본인인증 또는 간편 로그인은 확장 단계에서 도입
- 알림: 카카오 알림톡, 문자, 이메일, 앱 푸시 중 선택

### 6.3 데이터베이스

- PostgreSQL
- PostGIS 확장으로 공간 데이터 처리
- Redis 캐시
- OpenSearch 또는 Elasticsearch로 키워드 검색
- Vector DB: pgvector, Qdrant, Weaviate 중 선택

### 6.4 AI·검색

- 문서 요약 LLM
- 검색증강생성(RAG)
- 한국어 문서 임베딩 모델

## 7. 실시간 업데이트 설계

### 7.1 데이터별 갱신 주기

| 데이터 | 갱신 주기 | 이유 |
| --- | --- | --- |
| 교통통제 정보 | 5~10분 | 실시간 생활영향이 큼 |
| 달서구 고시공고 | 1~3시간 | 신규 공고와 마감일 반영 필요 |
| 보도자료·입법예고 | 1일 | 실시간성은 낮지만 정책 변경 확인 필요 |
| 구의회 회의록 | 1일 | 회의록 공개 주기 반영 |
| 공공시설·행정구역 데이터 | 월 1회 | 변경 빈도 낮음 |
| 주민 제보 | 실시간 | 민원성 정보 즉시 반영 필요 |

### 7.2 신규 데이터 처리 절차

1. 스케줄러가 데이터 출처별 수집 작업을 실행한다.
2. 게시글 ID, 제목, 작성일, 첨부파일 해시를 기존 데이터와 비교한다.
3. 신규 또는 수정 데이터만 파싱 큐에 넣는다.
4. 텍스트 추출 후 AI 분석 큐에 넣는다.
5. 주소·기간·영향유형을 추출하고 좌표화한다.
6. 검색 DB와 벡터 DB를 업데이트한다.
7. 관심 주소를 등록한 주민에게 영향권 일치 여부를 계산한다.
8. 알림 조건에 해당하면 요약 알림을 발송한다.

### 7.3 알림 조건

- 관심 주소 반경 500m 이내 신규 공고 등록
- 교통통제 시작 24시간 전, 1시간 전
- 의견 제출 마감 3일 전, 1일 전
- 주민설명회 개최 3일 전, 당일 오전
- 긴급 교통통제나 재난성 정보는 즉시 알림

## 10. 기대 효과

### 주민 측면

- 어려운 고시공고를 쉬운 말로 이해할 수 있다.
- 내 주소와 관련 있는 행정정보를 놓치지 않는다.
- 의견 제출 기한과 방법을 쉽게 확인할 수 있다.
- 공사, 통제, 정책 변경으로 인한 생활 불편을 미리 대비할 수 있다.

### 달서구청 측면

- 공고문을 읽지 못해 발생하는 반복 문의를 줄일 수 있다.
- 주민참여와 의견 제출률을 높일 수 있다.
- 행정정보 공개의 실질적 효과를 높일 수 있다.
- 정책 추진 과정에서 주민 신뢰도를 높일 수 있다.

### 공공서비스 혁신 측면

- 공개되어 있지만 활용하기 어려운 행정정보를 시민 맞춤형 정보로 전환한다.
- AI를 단순 상담 챗봇이 아니라 행정문서 해석, 공간 분석, 참여 지원 도구로 활용한다.
- 달서구의 지역 행정자료를 기반으로 한 맞춤형 서비스라는 점에서 지역성이 분명하다.

## 11. 차별성

기존 서비스는 고시공고, 회의록, 교통정보를 각각 따로 제공한다. 본 서비스는 이 정보를 통합하여 주민 주소 기준으로 재구성한다.

차별점:

- 행정기관 기준 정보가 아니라 주민 주소 기준 정보 제공
- 단순 검색이 아니라 생활영향 중심 요약
- 공고문, 회의록, 교통통제, 도시계획 정보를 통합 분석
- 주민 의견 제출까지 이어지는 참여형 서비스
- 달서구 행정자료를 활용한 지역 맞춤형 AI 공공서비스

## 12. 개인정보·보안 고려사항

- 주소는 관심 지역 분석 목적으로만 사용한다.
- 로그인 없이 1회성 주소 검색 기능을 기본 제공한다.
- 관심 주소 저장 시 최소한의 개인정보만 수집한다.
- 주민 제보 사진은 얼굴, 차량번호 등 개인정보 자동 마스킹을 적용한다.
- AI 학습에는 개인정보가 포함된 원문을 직접 사용하지 않는다.
- 공공데이터와 공개문서 원문 출처를 명확히 표시한다.

## 13. 리스크와 대응 방안

### 리스크 1. 공고문에 위치 정보가 명확하지 않은 경우

대응:

- 주소, 시설명, 행정동, 도로명, 지번을 모두 추출한다.
- 좌표 변환 실패 시 담당부서 확인 필요로 표시한다.
- 행정 담당자 검수 기능을 제공한다.

### 리스크 2. AI가 잘못 요약할 가능성

대응:

- 원문 근거 링크를 반드시 제공한다.
- 중요한 행정 절차와 마감일은 규칙 기반으로 별도 추출한다.
- 행정 담당자 검수 후 공개하는 운영 모드를 제공한다.

### 리스크 3. 실시간 데이터 품질 차이

대응:

- 데이터별 마지막 갱신 시각 표시
- API 장애 시 캐시 데이터 표시
- 출처별 신뢰도를 구분하여 표시

### 리스크 4. 주민 주소 저장에 대한 부담

대응:

- 로그인 없는 일회성 검색 제공
- 관심 주소는 별칭으로 저장 가능
- 정확한 상세주소 대신 도로명 또는 행정동 단위 알림 선택 가능

## 15. 공모전 제안서 핵심 문장

기존 행정정보는 이미 공개되어 있지만 주민은 그것이 내 생활과 어떤 관련이 있는지 알기 어렵다. 달서 AI 내 주소 영향 요약 서비스는 달서구의 고시공고, 도시계획, 교통 정보를 주민 주소 기준의 생활영향 정보로 바꾸고, 후속 단계에서 OpenAI 기반 채팅과 리포트로 판단 지원을 강화하는 시민 맞춤형 공공서비스이다.
