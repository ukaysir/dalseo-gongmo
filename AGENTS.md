## 1. 최종 목표

2026 달서 전국 대학생 AI활용 아이디어 콘테스트에 제출할 “달서 길눈AI” 아이디어 제안서를 완성한다.

이 작업의 목표는 실제 서비스를 개발하거나 AI 모델을 학습하는 것이 아니다. 심사위원이 문제의 지역성, AI 활용 필요성, 기존 서비스와의 차별성, 기대효과와 실현 가능성을 빠르게 이해하도록 다음 제출자료를 만드는 것이다.

- 검증된 수치와 공식 출처
- 제안서에 삽입할 정적 PNG UI 목업
- 서비스 흐름도와 전후 비교 도식
- 적합성·창의성·효과성 평가기준에 대한 최종 점검

실제 앱, 백엔드, 데이터베이스, API 연동, AI 모델, 로그인, 성능시험은 만들지 않는다. 기술 구현은 아이디어의 실현 가능성을 설명할 정도로만 작성한다.

## 4. 제안서 서식 구조

- 사회적 기대효과

한 문장 정의:

> 달서구의 보행로정보수집 사업 기반에서 촬영한 보행장벽 사진을 AI가 품질검수·분류·중복통합하고, 위험도에 따라 담당자의 보수 업무로 연결한 뒤 재촬영으로 처리결과까지 확인하도록 돕는 사람 중심의 보행안전 행정 아이디어다.

핵심 흐름:

현장 촬영 → AI 사진 품질검수 → 장애유형 후보 분류 → 동일지점 중복통합 → 설명 가능한 보수 우선순위 → 담당자 확인 → 작업카드와 상태추적 → 동일지점 재촬영 → 전후 검증

AI는 후보와 근거를 제시한다. 법적 판단, 보수 결정, 사건 종결은 사람이 최종 확인한다.

## 6. 차별화 기준

스마트폰으로 보행장벽 사진과 위치를 수집하는 기능은 이미 Road Scanner에 있다. 서울 동행맵은 시민 제보를 담당자가 확인하고 관련 부서에 전달하는 흐름도 공개했다. 따라서 아래 표현만으로는 창의성이 부족하다.

- 사진으로 보행장벽을 신고한다.
- 지도에 위험지점을 표시한다.
- 담당 부서에 전달한다.
- 교통약자에게 길을 안내한다.

길눈AI의 차별성은 다음 기능을 하나의 운영 흐름으로 묶는 데 있다.

1. 달서구 2026년 사업자료에 확인되는 177명 규모 현장수집 기반과의 연계
2. 고령 조사자를 고려한 큰 글씨·음성 중심 촬영 품질 안내
3. 장애유형 자동 후보 분류
4. 위치와 이미지 유사도를 이용한 동일지점 중복통합
5. 복지시설·정류장 인접성 등을 반영한 설명 가능한 우선순위
6. 구조화된 행정 작업카드와 처리상태 추적
7. 재촬영을 통한 보수 전후 검증
8. 사람의 최종 승인과 개인정보 최소화

“국내 최초”, “기존 유사 서비스가 없다”라는 표현은 사용하지 않는다.

## 8. 주장과 출처 검증 규칙

- 최신성이 필요한 수치·법령·서비스 기능은 인터넷에서 다시 확인한다.
- 공식기관, 원문 데이터, 원 논문, 서비스 개발자의 공식 설명을 우선한다.
- 검색결과 요약문만으로 수치를 확정하지 않는다.
- CSV 자료는 미리보기 일부가 아니라 전체 행을 계산한다.
- 합계는 계산식을 evidence.md에 기록한다.
- 제안서에는 안정적인 상세페이지 URL을 인용하고 일시적인 직접 다운로드 URL은 사용하지 않는다.
- 출처명, 자료명, 제공기관, 기준일, URL, 확인일을 기록한다.
- 서로 기준일이 다른 자료로 비율을 만들지 않는다.
- 선행연구의 정확도나 효과를 길눈AI가 달성한 수치처럼 쓰지 않는다.
- 서비스 소개문에 없는 기능을 “없다”고 단정하지 않고 “공개 소개문에서 확인되지 않음”으로 처리한다.

제안서의 모든 문장을 다음 세 종류로 구분해 작성한다.

1. 검증된 현황: 출처와 기준일 필수
2. 길눈AI 제안 기능: “제안한다”, “설계한다”, “활용한다”로 표현
3. 기대효과·목표: “기대한다”, “목표로 한다”, “실증으로 검증한다”로 표현

## 9. 사용 금지 또는 조건부 주장

다음 내용은 출처 또는 검증 없이 사실처럼 쓰지 않는다.

- 네 수행기관이 모두 같은 앱 또는 Road Scanner를 사용한다.
- 현재 수집자료가 보수 업무에 전혀 활용되지 않는다.
- 사진 반려율, 중복률, 수동입력 시간이 높다.
- 길눈AI가 업무시간을 특정 비율로 줄였다.
- 일반 스마트폰 사진 한 장으로 센티미터 단위 단차·경사를 정확히 측정한다.
- AI가 법 위반 여부를 자동 판정한다.
- 달서구 또는 수행기관이 실증에 동의했다.
- 서비스가 법률을 완전히 준수한다.

현재 업무의 문제를 단정하는 대신 다음처럼 쓴다.

> 기존 현장수집 기반에 AI 품질검수, 중복통합, 우선순위, 처리상태와 전후검증 기능을 더해 행정 활용성을 강화한다.

## 10. 정적 UI 목업 규칙

목업은 제안서 설명용 이미지이며 실제 작동 제품이 아니다.

권장 화면:

1. 현장촬영: 촬영 대상, 흔들림·가림 안내, 얼굴·번호판 흐림
2. AI 분석결과: 보도 파손·점자블록·통행 장애물 후보와 수정 버튼
3. 행정 대시보드: 예시 지도, 위험도, 중복 묶음, 상태 필터
4. 보수 작업카드: 위치, 근거, 담당 부서 후보, 담당자 승인
5. 전후 검증: 보수 전후 사진과 완료 확인

추가 도식:

- 수집부터 전후검증까지의 서비스 흐름도 1장
- 기존 흐름과 길눈AI 흐름 비교 1장

목업 제작 기준:

- HTML/CSS 등으로 정적 화면을 만든 뒤 PNG로 캡처해도 된다.
- 실제 클릭, 저장, 서버통신, 지도 API, AI 추론은 구현하지 않는다.
- 제안서에 삽입해도 읽히도록 큰 한글과 높은 대비를 사용한다.
- 실제 지도 대신 도식형 예시 지도를 사용하거나 지도 라이선스와 출처를 표시한다.
- 모든 가상 수치와 위치에는 “화면 예시”, “가상 데이터”, “AI 분석 예시” 중 하나를 표시한다.
- AI 결과에는 “담당자 확인 전” 상태를 표시한다.
- 177명, 111,111명, 28,256명처럼 검증된 수치만 실제 근거값으로 사용할 수 있다.
- 위험도, 신뢰도, 처리일, 완료율, 시간 감소율은 모두 예시값으로 표시한다.

정적 목업을 만들기 위해 애플리케이션 전체를 설계하거나 불필요한 코드 구조를 만들지 않는다. PNG 산출물의 시각적 전달력이 우선이다.

## 11. 기술·실현 가능성 서술 범위

제안서에는 아래 수준이면 충분하다.

- 컴퓨터 비전: 사진 품질 확인과 보행장벽 후보 분류
- 공간분석: GPS와 이미지 유사도를 이용한 중복통합
- 우선순위 엔진: 위험도와 주변 시설을 이용한 설명 가능한 점수
- 생성형 AI: 구조화된 작업카드와 보고문 초안
- 사람 검증: 최종 유형, 우선순위, 담당 부서와 완료상태 승인

기술스택, 모델명, 서버 사양, 데이터베이스 설계, API 명세는 제안서 이해에 꼭 필요한 경우가 아니면 넣지 않는다.

1차 범주는 다음 세 가지로 제한한다.

- 보도 파손·들뜸·단차 후보
- 점자블록 파손·가림
- 통행 방해 장애물

법정 단차·경사·폭 판정은 AI 후보 탐지와 사람의 추가 측정을 분리한다.

실증계획은 실현 가능성을 보여 주는 짧은 단계표로만 제시한다. 실제 실증을 수행한 것처럼 표현하지 않는다.

## 12. 개인정보·윤리 표현

목업과 본문에 다음 원칙을 포함한다.

- 연속 차량영상보다 목적이 분명한 단일 현장사진 우선
- 사람 없는 구도로 촬영 안내
- 얼굴·차량번호 업로드 전 흐림
- 원본 최소수집·최소보관
- 역할별 접근권한
- AI 저신뢰 결과의 재촬영 또는 사람 검토
- 단속·개인추적 목적 사용 금지
- 행정조치와 사건 완료는 사람이 최종 승인
- 실제 도입 전 개인정보·법률 검토

법적 적합성을 확정적으로 보증하지 않는다.

## 13. 제안서 작성 순서

1. evidence.md의 검증 문장으로 제안배경 초안을 작성한다.
2. 길눈AI의 한 문장 정의와 핵심 흐름을 고정한다.
3. Road Scanner와 서울 동행맵을 인정한 뒤 구체적인 차별점을 제시한다.
4. 네 개 서식 항목에 맞춰 proposal_draft.md를 작성한다.
5. 제안내용과 정확히 대응하는 정적 목업 5개와 흐름도를 제작한다.
6. 출처 각주와 기준일을 삽입한다.
7. 적합성·창의성·효과성 기준으로 자체 점검한다.
8. HWP 서식에 옮기기 전 문장, 표, 이미지 캡션을 최종 검수한다.

인터뷰는 있으면 보강자료가 되지만 제출의 필수조건으로 두지 않는다. 인터뷰가 없으면 공식 공개자료로 확인된 사실만 사용한다.

## 14. HWP와 HWPX 처리 규칙

현재 제공된 AI활용 아이디어 제안서_서식.hwp는 바이너리 HWP 파일이다.

- 원본 HWP는 읽기 참고만 하고 직접 수정하거나 생성하지 않는다.
- 우선 Markdown으로 원고를 완성하고 PNG 목업을 만든다.
- 사용자가 한글에서 원고와 이미지를 서식에 삽입할 수 있도록 전달한다.
- 사용자가 HWPX 파일을 제공하거나 HWPX 결과물을 명시적으로 허용한 경우에만 hwpx 스킬 지침에 따라 작업한다.
- HWPX 편집 시 원본 구조, 쪽수, 표와 스타일을 보존하고 필수 검증 도구를 통과해야 한다.

## 15. 완료 기준

다음 조건을 모두 만족해야 제안서 준비가 완료된 것으로 본다.

- 제안서 원고가 요지, 제안배경, 제안내용, 기대효과의 네 항목을 모두 채운다.
- 177명, 111,111명, 28,256명의 표현과 기준일이 evidence.md와 일치한다.
- 모든 현황 수치와 선행사례에 출처가 있다.
- 검증된 사실, 제안 기능, 기대효과가 문장상 구분된다.
- 기존 유사 서비스와 중복되는 기능을 인정하고 차별점이 명확하다.
- 실제 구현이나 실증을 완료한 것처럼 보이는 표현이 없다.
- 정적 PNG 목업에 가상 데이터 표시가 있다.
- AI의 역할과 사람의 최종 승인 흐름이 보인다.
- 개인정보 최소화 원칙이 포함된다.
- 적합성, 창의성, 효과성 각각에 대응하는 근거가 있다.
- 5분 안에 문제, 해결, 차별성, 효과를 설명할 수 있다.
- 제출 전 공식 마감 시각과 파일요건을 다시 확인한다.
