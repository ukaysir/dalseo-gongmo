## 3. 그림별 요구사항 검증

### 그림 1 — 전체 운영 흐름도

- 177명 옆에 배치 인원 합계 각주를 표시했다.
- 우측 하단에 `177명: 2026년 사업자료상 4개 수행기관 배치 인원 합계`와 자료명을 표시했다.
- AI 품질검수·유형 후보·동일지점 통합·조치 우선순위를 표시했다.
- 관리주체 확인 뒤 직접 조치·현장 실측·타기관 이첩·장기검토로 분기했다.
- `규칙 기반 후보 · 담당자 확정`을 표시했다.
- 모든 처리경로 뒤 동일지점 재촬영과 조치 전후검증이 이어지는 구조를 확인했다.
- 의미가 불명확했던 관리주체 분기 상단의 사람 모양 마름모 아이콘을 삭제했다.
- 확정 슬로건 외 상위 처리용어는 `조치`로 통일했다.

판정: **통과**

### 그림 2 — 현장촬영·AI 검수 화면

- 안전경로 대신 조사구역과 조사순서를 표시했다.
- 실제 지도 대신 직접 제작한 도식형 SVG 지도를 사용했다.
- 이전 사진 반투명 윤곽, 촬영방향, 대상 위치 가이드와 GPS 오차 안내를 표시했다.
- 큰 글씨·음성 재촬영 안내를 표시했다.
- AI 유형 후보를 조사자가 확인·수정·재촬영하는 정적 버튼을 표시했다.
- 중앙 파손부가 크게 보이는 생성 사진을 촬영 미리보기와 AI 후보 화면에 동일하게 사용했다.
- 탐지 프레임이 실제 점자블록 파손·탈락 영역을 감싸도록 좌표를 조정했다.
- 가상 위치, AI 분석 예시, 담당자 확인 전, 법정치수 판단 아님을 표시했다.
- 1차 캡처에서 확인된 촬영 버튼 잘림을 수정하고 재캡처했다.

판정: **통과**

### 그림 3 — 행정 작업카드·전후검증

- 사건번호·가상 위치·유형 후보와 현재 상태를 표시했다.
- 우선순위의 네 근거를 숫자 총점 없이 항목별로 표시했다.
- 위치·시설유형·관리대장·업무규칙을 입력으로 하는 규칙 기반 분기를 표시했다.
- 생성형 AI 역할을 확정된 정보의 조치요청문 초안 작성으로 제한했다.
- 같은 지점·같은 구도의 생성 사진으로 점자블록 파손 전과 보수 후를 비교했다.
- 파손부를 중앙에 크게 배치하고 전·후 후보 프레임을 같은 좌표에 정렬했다.
- 사건 요약 카드 높이와 내부 셀 높이를 조정해 상태 배지가 테두리 안에 유지되도록 했다.
- 완료·재조치·타기관 확인 선택과 담당자 최종 승인을 표시했다.
- 접수부터 재확인까지의 가상 처리이력 타임라인을 표시했다.
- 실제 주민 공개화면과 실제 부서·사건정보는 사용하지 않았다.

판정: **통과**
